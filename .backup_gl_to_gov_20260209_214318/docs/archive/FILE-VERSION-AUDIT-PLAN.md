# 文件版本核對與升級計劃

## 任務目標
系統性核對所有文件與 main 最新版本的差異，制定升級路線

## 核對策略

### 1. 分類核對
- 配置文件（YAML/YML）
- 文檔文件（MD）
- 代碼文件（Python/TypeScript）
- 執行腳本（Shell）

### 2. 版本比對
- 本地工作區文件
- Git 倉庫中的最新版本
- 其他分支中的更新版本

### 3. 差異分析
- 內容差異
- 結構差異
- 版本號差異
- 時間戳差異

## 執行步驟

### Phase 1: 文件清單生成
- [ ] 掃描所有文件
- [ ] 按類型分類
- [ ] 記錄文件元數據

### Phase 2: 版本比對
- [ ] 比對工作區 vs Git
- [ ] 比對各分支間差異
- [ ] 識別最新版本

### Phase 3: 差異分析
- [ ] 記錄具體差異
- [ ] 評估差異影響
- [ ] 判斷正確版本

### Phase 4: 升級計劃制定
- [ ] 制定每個文件的升級策略
- [ ] 確定升級優先級
- [ ] 制定執行順序

### Phase 5: 執行升級
- [ ] 按計劃執行升級
- [ ] 驗證升級結果
- [ ] 提交更改

## 核對工具
- git diff
- git log
- git show
- file 命令
- md5sum/sha256sum
